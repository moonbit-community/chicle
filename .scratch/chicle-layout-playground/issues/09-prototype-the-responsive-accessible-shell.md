# Prototype the responsive accessible shell

Type: prototype
Status: resolved
Blocked by: 06, 07

## Question

How should the editor, preview, examples, errors, and sharing controls behave on desktop and narrow screens while remaining keyboard-usable, readable, and visually simple?

## Answer

Desktop uses balanced editor and preview panels. Below 980 px the panels stack, and below 620 px controls and footers reflow vertically. Controls have accessible names and visible keyboard focus; status and errors use live text rather than color alone.
