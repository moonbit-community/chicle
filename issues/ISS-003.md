# ISS-003: Preserve layout metadata when offsetting a subtree

## Metadata
- Type: bug
- Status: in_progress
- Priority: 0
- Labels: tree, layout-metadata, relative-positioning, margin-collapse, regression, tests
- Assignee: unassigned
- Created: 2026-07-12
- Updated: 2026-07-12
- External ref: none

## Description
`offset_subtree` rebuilds every moved node from `Layout::zero()` and restores only `location` and `size`. This discards previously computed `content_size`, `scrollbar_size`, `border`, `padding`, and `order` values. The caller may re-finalize the subtree root, but descendants are not generally re-finalized.

This is confirmed at runtime. A padded leaf reports padding `3,4,5,6` in the control tree. Placing its parent container at a relative inset of 10px causes the same leaf to report padding `0,0,0,0`, although its style and size are unchanged. The affected helper is also used for block margin-collapse displacement, so the impact is not limited to explicit relative positioning.

## Design
Offset layouts by copying the complete existing `Layout` and replacing only `location`. Preserve every other metadata field exactly. Audit all callers to ensure they intend to translate the entire subtree and do not depend on accidental zeroing. Add regression tests for nested relative positioning and block margin-collapse adjustment. The fix should remain surgical and must not trigger layout recomputation.

## Acceptance Criteria
- [ ] `offset_subtree` changes only each live node's location and preserves size, content size, scrollbar size, border, padding, and order.
- [ ] A padded descendant under a relatively offset container retains padding `3,4,5,6` after layout.
- [ ] Nested scrollbar and content-size metadata survive subtree translation.
- [ ] Block margin-collapse displacement preserves all layout metadata.
- [ ] Focused downstream-style regression tests fail before the fix and pass after it.
- [ ] `moon check`, `moon check --warn-list +unnecessary_annotation`, and `moon test --no-render --target all` pass.
